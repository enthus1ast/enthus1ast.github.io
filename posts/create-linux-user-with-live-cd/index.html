Create Linux User With Live CD

By David Krause (enthus1ast)

July 6, 2023

Today i had to fix a linux laptop, that did not boot correctly, and where i did not know the password of.

The graphical user interface did not came up because of a full ssd. This i did not know yet when i got the laptop. I figured that it would be easiest to login on the system to have easy access to journalctl etc.

So i created a user from a live linux CD called GRML. But this works from most linux live distros.

  1. Boot into a live cd, in my case i used GRML which i always carry around on a usb stick on my keychain.

  2. mount the root partition of the linux notebook

mkdir /mnt/sda1
mount /dev/sda1 /mnt/sda1
  1. Create a user (i chose: admin) on the livecd with:
useradd admin
  1. Set a password for user admin (still on the livecd)
passwd admin
  1. create the user on the target system

step 3 and 4 will create an entry on the livecd’s /etc/passwd and /etc/shadow, these entries we will change a little and copy to the other linux.

cat /etc/passwd
# ...
admin:x:1001:1001::/home/admin:/bin/sh

cat /etc/shadow
# ---
admin:$y$j9T$tHOGMhz52m.AciF0XasKc/$.XPZ8p0Jord0G.Z3bze8fAH/4GmmEWof3ZDnaEuI7b3:19544:0:99999:7:::

we now append these lines to the to the system we wanna log in but we change the userid and group id of the newly created user to 0 (0 is the root user) :

# here we changed 1001 to 0
# and also changed the home dir `/home/admin/` to `/root`
echo "admin:x:0:0::/root:/bin/sh" >> /mnt/sda1/passwd

# this is user: `admin` password: `123`
echo "admin:$y$j9T$tHOGMhz52m.AciF0XasKc/$.XPZ8p0Jord0G.Z3bze8fAH/4GmmEWof3ZDnaEuI7b3:19544:0:99999:7:::" >> /mnt/sda1/shadow
  1. DONE Login on the target system

Reboot into the other linux system, and login with the user and password you’ve just created.

If you copied the commands above, the logins are:

Username: admin
Password: 123
